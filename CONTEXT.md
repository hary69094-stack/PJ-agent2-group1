# CONTEXT.md — 带长期记忆的对话型 LLM Agent

## 核心问题
如何让 LLM Agent 像人类一样「有选择性地记忆与遗忘」，而非简单地将所有历史对话存入向量库做全量检索。

## 术语定义

| 术语 | 定义 |
|------|------|
| **短期记忆（Short-Term Memory） | Agent 在单次会话中的上下文窗口内直接可见的近期对话历史，容量受 token 上限约束 |
| **长期记忆（Long-Term Memory） | 跨越多次会话边界的持久化知识存储，Agent 根据重要性判定选择性地写入、读取和遗忘 |
| **记忆选择（Memory Selection） | Agent 主动判断「什么信息值得长期记住」的决策过程，是选择性长期记忆的核心机制 |
| **记忆遗忘（Memory Forgetting） | Agent 根据规则或学习主动删除/淘汰长期记忆中不再需要的信息 |
| **一致性（Consistency） | 在多轮对话中，Agent 对同一事实的回答保持前后一致的程度 |
| *遗忘率（Forgetting Rate） | Agent 无法正确回忆已存储信息中目标事实的比率 |
| **记忆单元（Memory Unit） | 长期记忆中存储的最小信息单位，本系统采用结构化三元组 `{主语, 关系, 宾语}` |
| **重要性分数（Importance Score） | 记忆写入时由 LLM 分配 1-5 的评分，用于淘汰优先级判定 |
| **去重（Deduplication） | 防止语义相同或高度重复的三元组被重复写入记忆库的过滤机制 |
| **检索触发（Retrieval Trigger） | 决定 Agent 是否访问长期记忆库的判据：召回信号词检测 + 每 5 轮兜底触发 |
